# Карточка локальной модели

## Основной профиль

В проекте выбран локальный профиль **Qwen 3.5 (14B)**.

**Рекомендуемое квантование:** **Q4_K_M**.

| Характеристика | Значение |
| --- | --- |
| Название профиля | Qwen 3.5 (14B) |
| Назначение | диалог и генерация ответов в RAG |
| Количество параметров | 14B |
| Формат | GGUF |
| Квантование | Q4_K_M |
| Размещение | локальный том с доступом только для чтения |
| Сетевой доступ во время инференса | не требуется |
| Имя артефакта | `qwen3.5-14b-q4_k_m.gguf` |
| Публичный псевдоним API | `qwen3.5-14b-q4_k_m` |

## Почему выбрано Q4_K_M

Q4_K_M уменьшает размер 14B-модели и требования к памяти, сохраняя практичный уровень качества для
диалога и RAG. Это рекомендуемый профиль проекта для локальных рабочих станций. Фактическую скорость,
потребление RAM/VRAM и качество необходимо измерять на целевом оборудовании и продуктовых данных.

## Размещение артефакта

Рекомендуемая структура:

```text
models/
└── generator/
    └── qwen3.5-14b-q4_k_m.gguf
```

Каталог `models/` исключён из Git. В контейнере файл подключается отдельным томом только для чтения.

Для каждого выпуска необходимо отдельно сохранить в реестре артефактов:

- точный источник исходных весов;
- неизменяемую ревизию;
- инструмент и версию конвертации в GGUF;
- инструмент и параметры квантования Q4_K_M;
- SHA-256 итогового файла;
- лицензию и условия распространения;
- дату и результаты проверки качества.

## Совместимость рантайма

Q4_K_M является квантованием GGUF и требует совместимого локального рантайма, например llama.cpp.
Рантайм подключается к приложению через порт `ChatModel`. Базовый `HuggingFaceChatModel` использует
Transformers и предназначен для директорий с Transformers-чекпойнтом; он не должен получать путь к
GGUF-файлу.

Обозначение «Qwen 3.5 (14B)» в документации является именем зафиксированного модельного профиля этого
проекта. Источник и контрольная сумма конкретного 14B-артефакта должны быть заполнены владельцем
развёртывания до производственного запуска.

## Параметры генерации

Базовые серверные параметры:

```dotenv
MODEL__ALIAS=qwen3.5-14b-q4_k_m
MODEL__MAX_INPUT_TOKENS=4096
MODEL__MAX_NEW_TOKENS=512
MODEL__TEMPERATURE=0.2
MODEL__TOP_P=0.9
MODEL__DO_SAMPLE=false
MODEL__REPETITION_PENALTY=1.05
MODEL__CONCURRENCY=1
MODEL__TIMEOUT_SECONDS=120
```

Эти значения являются отправной точкой. Изменение контекста, шаблона диалога или параметров генерации
требует повторного запуска оценок диалога, RAG и безопасности.

## Приёмочная проверка

Перед развёртыванием:

1. Проверьте SHA-256 и права доступа к GGUF-файлу.
2. Убедитесь, что инференс не обращается к сети.
3. Проверьте русский и английский диалог.
4. Выполните продуктовые RAG-сценарии и проверьте ссылки на источники.
5. Проверьте отказ при отсутствии релевантного контекста.
6. Выполните сценарии внедрения инструкций.
7. Измерьте задержку, пропускную способность, RAM и VRAM.
8. Сохраните версию рантайма и результаты измерений вместе с модельным артефактом.
