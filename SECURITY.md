# Политика безопасности

Сообщайте о предполагаемых уязвимостях приватно через GitHub Security Advisories этого репозитория.
Не создавайте публичные issues с описанием эксплуатации, содержимым документов, приватными URL или
учётными данными развёртывания.

Поддерживается только последняя версия проекта. Владелец каждого развёртывания самостоятельно отвечает
за аутентификацию, авторизацию, сетевые ограничения, проверку локального артефакта Qwen 3.5 (14B),
хранение данных и управление доступом.

Перед публикацией API выполните `make security` и ознакомьтесь с
[моделью угроз](docs/security.md). Профиль Q4_K_M должен поступать из проверенного источника, иметь
зафиксированную контрольную сумму и подключаться к контейнеру только для чтения.
