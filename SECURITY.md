# Security policy

Please report suspected vulnerabilities privately through GitHub Security Advisories for this
repository. Do not open a public issue containing exploit details, document contents, private URLs, or
deployment credentials.

Only the latest revision of the template is maintained. A project generated from the template owns its
deployment-specific authentication, authorization, network, model, and data-retention controls.

Run `make security` for the dependency audit and consult [docs/security.md](docs/security.md) before
exposing the API outside a trusted network.
